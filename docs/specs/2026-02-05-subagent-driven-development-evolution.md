# Subagent-Driven Development Skill Evolution Spec

## 1. 背景与目标 (Context & Goals)
- **痛点**: 子代理委派流程缺少明确的握手协议与零信任审计步骤，JSON 报告字段不完整。
- **目标**: 明确委派协议、隔离边界、审计流程与 JSON 汇报字段。
- **非目标 (Non-Goals)**: 不设计新的工具或自动化系统。

## 2. 核心设计 (Core Design)
- **API 变更**: 定义“握手协议”与“零信任审计”步骤。
- **数据结构**: JSON 汇报格式包含文件清单、验证日志、风险与遗漏。
- **逻辑流**: 委派 -> 执行 -> 双审查 -> 汇总。

## 3. 方案权衡 (Trade-offs)
| 维度 | 方案 A (推荐) | 方案 B (替代) |
| :--- | :--- | :--- |
| **复杂度** | 中 | 低 |
| **风险** | 低：角色清晰，可审计 | 中：混合角色导致遗漏 |
| **维护成本** | 中 | 低 |

## 4. 风险与副作用 (Risks & Side Effects)
- **受影响文件**: `skills/subagent-driven-development/SKILL.md`
- **破坏性变更**: 交互更繁琐但可靠性提高。
