# UX-Prototyping Skill Evolution Spec

## 1. 背景与目标 (Context & Goals)
- **痛点**: UX 原型规范偏模板化，缺少“中文友好”和“单文件交付”强约束。
- **目标**: 强化单文件交付、中文文案、交互验证与禁用复杂构建工具。
- **非目标 (Non-Goals)**: 不要求生产级视觉或性能优化。

## 2. 核心设计 (Core Design)
- **API 变更**: 强制单文件 HTML 模板与中文文案规范。
- **数据结构**: 原型包含关键交互路径与错误状态。
- **逻辑流**: 需求澄清 -> 草图 -> 单文件原型 -> 交互验证。

## 3. 方案权衡 (Trade-offs)
| 维度 | 方案 A (推荐) | 方案 B (替代) |
| :--- | :--- | :--- |
| **复杂度** | 中 | 低 |
| **风险** | 低：快速验证 | 中：需求偏差 |
| **维护成本** | 中 | 低 |

## 4. 风险与副作用 (Risks & Side Effects)
- **受影响文件**: `skills/ux-prototyping/SKILL.md`
- **破坏性变更**: 视觉探索空间减少。
