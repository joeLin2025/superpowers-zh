# Writing-Skills Skill Evolution Spec

## 1. 背景与目标 (Context & Goals)
- **痛点**: 技能编写缺少可执行文档与验收标准，TDD 流程不够明确。
- **目标**: 强化“先红后绿”与验收标准，输出可执行的技能模板。
- **非目标 (Non-Goals)**: 不设计新的工具。

## 2. 核心设计 (Core Design)
- **API 变更**: 增加验收标准与测试案例要求。
- **数据结构**: 技能文档必须包含可验证的规则。
- **逻辑流**: 观测失败 -> 编写规则 -> 验证规则 -> 打磨。

## 3. 方案权衡 (Trade-offs)
| 维度 | 方案 A (推荐) | 方案 B (替代) |
| :--- | :--- | :--- |
| **复杂度** | 中 | 低 |
| **风险** | 低：可执行 | 中：空洞规则 |
| **维护成本** | 中 | 低 |

## 4. 风险与副作用 (Risks & Side Effects)
- **受影响文件**: `skills/writing-skills/SKILL.md`
- **破坏性变更**: 文档更严格，编写耗时增加。
