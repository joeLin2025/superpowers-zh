# Brainstorming Skill Evolution Spec

## 1. 背景与目标 (Context & Goals)
- **痛点**: 现有 `skills/brainstorming/SKILL.md` 对“输出规格化”与“反向压力测试”的要求不够明确，且缺少与 ADR/设计思维框架的对齐。
- **目标**: 引入标准化 Spec/ADR 结构、强制双方案权衡、反向压力测试与风险矩阵，确保脑暴结果可执行、可审计。
- **非目标 (Non-Goals)**: 不改动其他技能；不引入实现代码。

## 2. 核心设计 (Core Design)
- **API 变更**: 固化“路由前置审计模板”和“Spec/ADR 输出模板”。
- **数据结构**: 增加设计思维五阶段提示（同理-定义-构思-原型-测试）与 ADR 记录字段。
- **逻辑流**: Context Audit -> 双方案 -> 反向压力测试 -> Spec/ADR 输出 -> 用户确认。

## 3. 方案权衡 (Trade-offs)
| 维度 | 方案 A (推荐) | 方案 B (替代) |
| :--- | :--- | :--- |
| **复杂度** | 中 | 低 |
| **风险** | 低：结构化输出，误解风险可控 | 中：输出碎片化，难以审计 |
| **维护成本** | 中：模板需维护 | 低：文档短但质量不稳定 |

## 4. 风险与副作用 (Risks & Side Effects)
- **受影响文件**: `skills/brainstorming/SKILL.md`
- **破坏性变更**: 输出更冗长；需要更多时间完成脑暴。
