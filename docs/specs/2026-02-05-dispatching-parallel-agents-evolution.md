# Dispatching-Parallel-Agents Skill Evolution Spec

## 1. 背景与目标 (Context & Goals)
- **痛点**: 并行调度缺少明确的拆解准则与合并冲突预防措施。
- **目标**: 引入 Map/Reduce 思路、任务隔离清单与合并前全量回归要求。
- **非目标 (Non-Goals)**: 不引入自动化合并或 CI 系统。

## 2. 核心设计 (Core Design)
- **API 变更**: 增加“任务拆解准则”和“合并前校验”。
- **数据结构**: 每个子任务必须声明修改文件清单与验证方法。
- **逻辑流**: 拆解 -> 冲突预检 -> 并行执行 -> 合并审计 -> 全量回归。

## 3. 方案权衡 (Trade-offs)
| 维度 | 方案 A (推荐) | 方案 B (替代) |
| :--- | :--- | :--- |
| **复杂度** | 中 | 低 |
| **风险** | 低：冲突可控 | 中：隐性耦合爆炸 |
| **维护成本** | 中 | 低 |

## 4. 风险与副作用 (Risks & Side Effects)
- **受影响文件**: `skills/dispatching-parallel-agents/SKILL.md`
- **破坏性变更**: 并行调度准备时间增加。
