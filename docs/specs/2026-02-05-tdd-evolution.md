# TDD Skill Evolution Spec

## 1. 背景与目标 (Context & Goals)
- **痛点**: 现有 TDD 规则对测试异味、反模式约束不够清晰，容易出现“假绿”与补测。
- **目标**: 引入测试异味清单、质量门禁与红-绿-重构流程约束，明确禁止补测。
- **非目标 (Non-Goals)**: 不引入具体测试框架或工具配置。

## 2. 核心设计 (Core Design)
- **API 变更**: 增加“测试异味清单”和“质量门禁”。
- **数据结构**: 输出必须包含红-绿-重构三阶段检查表。
- **逻辑流**: Red -> Green -> Refactor -> Quality Gate -> Proceed.

## 3. 方案权衡 (Trade-offs)
| 维度 | 方案 A (推荐) | 方案 B (替代) |
| :--- | :--- | :--- |
| **复杂度** | 中 | 低 |
| **风险** | 低：可验证，假绿减少 | 中：短期快但易漏测 |
| **维护成本** | 中 | 低 |

## 4. 风险与副作用 (Risks & Side Effects)
- **受影响文件**: `skills/test-driven-development/SKILL.md`
- **破坏性变更**: 初期开发节奏变慢但更稳定。
