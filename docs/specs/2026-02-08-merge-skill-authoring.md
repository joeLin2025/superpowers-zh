# Merge Skill Authoring Skills

## 1. 背景与目标 (Context & Goals)
- **痛点**:
  - `writing-skills` 与 `skill-localization-and-polishing` 职责边界部分重叠（规则可执行性、对抗测试、文案约束）。
  - 用户触发路由时需要区分“新建/重构技能”与“本地化/打磨”，增加认知成本。
- **目标**:
  - 降低技能路由复杂度，保留强约束与可审计流程。
  - 保持规则可执行、可验证、可审计，不削弱现有门禁。
- **非目标 (Non-Goals)**:
  - 不改动现有技能之外的工程流程。
  - 不引入新工具或外部依赖。

## 2. 核心设计 (Core Design)
- **API 变更**:
  - 选项 A：合并为单一技能 `skill-authoring`（暂定名），更新 `using-superpowers` 路由关键词指向新技能。
  - 选项 B：保留两技能，但抽取“通用约束模块”并引用到两者（保持分工）。
- **数据结构**:
  - 维持现有 SKILL.md 模板结构；新增/复用对抗测试与验证条款。
- **流程/伪代码**:
  - A: 路由 -> 统一技能 -> 分支流程（创建/重构/本地化） -> 产出/验证。
  - B: 路由 -> 两技能之一 -> 引用共享条款 -> 产出/验证。

## 3. 方案权衡 (Trade-offs)
| 维度 | 方案 A (推荐：合并为单技能) | 方案 B (替代：保留分工) |
| :--- | :--- | :--- |
| 复杂度 | 路由更简单，但单技能内容更长 | 路由较复杂，但技能更聚焦 |
| 风险 | 合并时可能丢失细节或弱化约束 | 维护成本高，重复条款易漂移 |
| 维护成本 | 单点维护，更新更集中 | 多点维护，需要同步约束 |
| 性能影响 | 无 | 无 |

## 4. 风险与副作用 (Risks & Side Effects)
- **受影响文件**:
  - `skills/writing-skills/SKILL.md`
  - `skills/skill-localization-and-polishing/SKILL.md`
  - `skills/using-superpowers/SKILL.md`
- **破坏性变更**:
  - 合并会改变路由关键词与技能名称，需同步更新所有引用。
