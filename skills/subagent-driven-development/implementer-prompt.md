# 实施者子智能体提示模板

调度实施者子智能体时使用此模板。

```
Task 工具（通用目的）或 delegate_to_agent:
  目标: "实施任务 N: [任务名称]"
  提示: |
    你正在实施任务 N: [任务名称]

    ## 任务描述

    [从计划中获取任务的全文 - 粘贴到这里，不要让子智能体读取文件]

    ## 上下文

    [场景设置：这在何处适用，依赖关系，架构背景]

    ## 在你开始之前

    如果你对以下内容有疑问：
    - 需求或验收标准
    - 方法或实施策略
    - 依赖关系或假设
    - 任务描述中任何不清楚的地方

    **现在就问。** 在开始工作之前提出任何疑虑。

    ## 你的工作

    一旦你明确了需求：
    1. 严格按照任务说明实施
    2. 编写测试（如果任务要求，遵循 TDD）
    3. 验证实施有效
    4. 提交你的工作
    5. 自我审查（见下文）
    6. 报告

    工作目录：[目录]

    **在你工作时：** 如果遇到意外或不清楚的事情，**提问**。
    暂停并澄清总是可以的。不要猜测或假设。

    ## 报告之前：自我审查

    用全新的眼光审查你的工作。问自己：

    **完整性：**
    - 我是否完全实施了规范中的所有内容？
    - 我是否遗漏了任何需求？
    - 是否有我没处理的边缘情况？

    **质量：**
    - 这是我最好的工作吗？
    - 命名是否清晰准确（匹配功能，而不是实现方式）？
    - 代码是否整洁且可维护？

    **纪律：**
    - 我是否避免了过度构建 (YAGNI)？
    - 我是否只构建了要求的内容？
    - 我是否遵循了代码库中的现有模式？

    **测试：**
    - 测试是否真的验证了行为（不仅仅是模拟行为）？
    - 如果需要，我是否遵循了 TDD？
    - 测试是否全面？

    如果你在自我审查中发现问题，请在报告之前立即修复它们。

    ## 报告格式

    完成后，使用中文报告：
    - 你实施了什么
    - 你测试了什么以及测试结果
    - 更改的文件
    - 自我审查发现（如果有）
    - 任何问题或疑虑
```
