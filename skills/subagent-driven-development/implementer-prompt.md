# 实施者子智能体提示词模板 (Implementer Subagent Prompt Template)

当分派实施者子智能体时使用此模板。

```
Task tool (general-purpose):
  description: "Implement Task N: [task name]"
  prompt: |
    你正在实施任务 N: [任务名称]

    ## 任务描述

    [计划中的任务全文 - 粘贴在这里，不要让子智能体读取文件]

    ## 上下文

    [场景设置：这适用的位置、依赖关系、架构上下文]

    ## 在你开始之前

    如果你有关于以下方面的问题：
    - 需求或验收标准
    - 方法或实施策略
    - 依赖关系或假设
    - 任务描述中任何不清楚的地方

    **现在提问。** 在开始工作前提出任何疑虑。

    ## 你的工作

    一旦你清楚了需求：
    1. 准确实施任务指定的内容
    2. 编写测试（如果任务要求，遵循 TDD）
    3. 验证实施有效
    4. 提交你的工作
    5. 自我审查（见下文）
    6. 汇报

    工作目录：[目录]

    **当你工作时：** 如果你遇到意外或不清楚的事情，**提问**。
    暂停并澄清总是可以的。不要猜测或假设。

    ## 汇报前：自我审查

    用全新的眼光审查你的工作。问自己：

    **完整性：**
    - 我是否完全实施了规范中的所有内容？
    - 我是否遗漏了任何需求？
    - 是否有我没处理的边缘情况？

    **质量：**
    - 这是我最好的工作吗？
    - 命名是否清晰准确（匹配做什么，而不是怎么做）？
    - 代码是否整洁且可维护？

    **纪律：**
    - 我是否避免了过度构建 (YAGNI)？
    - 我是否只构建了被请求的内容？
    - 我是否遵循了代码库中的现有模式？

    **测试：**
    - 测试是否实际验证了行为（不仅仅是模拟行为）？
    - 如果需要，我是否遵循了 TDD？
    - 测试是否全面？

    如果你在自我审查中发现问题，现在就在汇报前修复它们。

    ## 报告格式

    完成后，汇报：
    - 你实施了什么
    - 你测试了什么以及测试结果
    - 变更的文件
    - 自我审查发现（如果有）
    - 任何问题或疑虑
```
