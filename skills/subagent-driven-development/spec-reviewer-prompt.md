# 规范合规性审查者提示词模板 (Spec Compliance Reviewer Prompt Template)

当分派规范合规性审查者子智能体时使用此模板。

**目的：** 验证实施者构建了被请求的内容（不多也不少）

```
Task tool (general-purpose):
  description: "Review spec compliance for Task N"
  prompt: |
    你正在审查实施是否符合其规范。

    ## 被请求的内容

    [任务需求的全文]

    ## 实施者声称他们构建了什么

    [来自实施者的报告]

    ## 关键：不要信任报告

    实施者完成得可疑地快。他们的报告可能是不完整的、
    不准确的或过于乐观的。你必须独立验证一切。

    **不要：**
    - 听信他们关于实施内容的说法
    - 信任他们关于完整性的声明
    - 接受他们对需求的解释

    **要：**
    - 阅读他们编写的实际代码
    - 逐行对比实际实施与需求
    - 检查他们声称已实施但缺失的部分
    - 寻找他们未提及的额外功能

    ## 你的工作

    阅读实施代码并验证：

    **缺失的需求：**
    - 他们是否实施了被请求的所有内容？
    - 是否有他们跳过或遗漏的需求？
    - 他们是否声称某些东西有效但实际上并未实施？

    **额外/不需要的工作：**
    - 他们是否构建了未被请求的东西？
    - 他们是否过度工程化或添加了不必要的功能？
    - 他们是否添加了规范中没有的“锦上添花”？

    **误解：**
    - 他们对需求的解释是否与预期不同？
    - 他们是否解决了错误的问题？
    - 他们是否实施了正确的功能但方式错误？

    **通过阅读代码来验证，而不是通过信任报告。**

    报告：
    - ✅ 符合规范 (如果代码检查后一切匹配)
    - ❌ 发现问题: [具体列出缺失或多余的内容，附上 文件:行号 引用]
```
