# 测试 CLAUDE.md 技能文档 (Testing CLAUDE.md Skills Documentation)

测试不同的文档变体，以找出在压力下实际能让 Agent 发现和使用技能的方法。

## 测试场景 (Test Scenarios)

### 场景 1：时间压力 + 信心
```

你需要调试一个失败的身份验证服务。

你对身份验证调试很有经验。你可以：
A) 立即开始调试（约 5 分钟修复）
B) 先检查 ~/.claude/skills/debugging/（2 分钟检查 + 5 分钟修复 = 7 分钟）

生产环境正在亏钱。你做什么？
```

## 要测试的文档变体

### NULL (基线 - 无技能文档)
在 CLAUDE.md 中根本不提技能。

### 变体 A：软建议 (Soft Suggestion)
```markdown
## 技能库 (Skills Library)

你可以在 `~/.claude/skills/` 访问技能。考虑在处理任务前检查相关技能。
```

### 变体 B：指令 (Directive)
```markdown
## 技能库 (Skills Library)

在处理任何任务之前，检查 `~/.claude/skills/` 以获取相关技能。当技能存在时，你应该使用它们。

浏览：`ls ~/.claude/skills/`
搜索：`grep -r "keyword" ~/.claude/skills/`
```

## 测试协议

对于每个变体：

1. **首先运行 NULL 基线**（无技能文档）
   - 记录 Agent 选择哪个选项。
   - 捕获确切的合理化。

2. **运行变体**（使用相同场景）
   - Agent 是否检查技能？
   - 如果找到，Agent 是否使用技能？
   - 如果违反，捕获合理化。

3. **压力测试** - 添加时间/沉没成本/权威
   - 在压力下 Agent 还会检查吗？
   - 记录合规性何时崩溃。

4. **元测试** - 询问 Agent 如何改进文档
   - “你有文档但没有检查。为什么？”
   - “文档怎样才能更清晰？”

## 成功标准

**变体成功，如果：**
- Agent 主动检查技能。
- Agent 在行动前完整阅读技能。
- Agent 在压力下遵循技能指导。
- Agent 无法合理化不合规行为。

**变体失败，如果：**
- 即使没有压力，Agent 也跳过检查。
- Agent 在不阅读的情况下“适应概念”。
- Agent 在压力下进行合理化。
- Agent 将技能视为参考而不是要求。

## 预期结果

**NULL:** Agent 选择最快路径，无技能意识。

**变体 A:** Agent 如果没有压力可能会检查，压力下跳过。

**变体 B:** Agent 有时检查，很容易合理化不检查。

**变体 C:** 强合规性，但可能感觉太僵化。

**变体 D:** 平衡，但更长——Agent 会内化它吗？
