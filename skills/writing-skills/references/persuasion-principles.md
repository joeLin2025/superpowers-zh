# 技能设计的说服原则 (Persuasion Principles for Skill Design)

## 概述

LLM 对说服原则的反应与人类相同。理解这种心理学有助于你设计更有效的技能——不是为了操纵，而是为了确保即使在压力下也能遵循关键实践。

**研究基础：** Meincke et al. (2025) 对 28,000 次 AI 对话测试了 7 个说服原则。说服技术使合规率增加了一倍多（33% -> 72%, p < .001）。

## 七个原则 (The Seven Principles)

### 1. 权威 (Authority)
**它是什么：** 对专业知识、证书或官方来源的尊重。

**它如何在技能中工作：**
- 祈使语言："YOU MUST"（你必须），"Never"（从不），"Always"（总是）。
- 不可商量的框架："No exceptions"（没有例外）。
- 消除决策疲劳和合理化。

**何时使用：**
- 纪律执行技能（TDD，验证要求）。
- 安全关键实践。
- 已建立的最佳实践。

**示例：**
```markdown
✅ 测试前写代码？删除它。重新开始。没有例外。
❌ 考虑在可行时先写测试。
```

### 2. 承诺 (Commitment)
**它是什么：** 与先前的行动、声明或公开宣言保持一致。

**它如何在技能中工作：**
- 要求宣布："Announce skill usage"（宣布技能使用）。
- 强制明确选择："Choose A, B, or C"（选择 A、B 或 C）。
- 使用跟踪：清单的 TodoWrite。

**何时使用：**
- 确保技能实际上被遵循。
- 多步骤流程。
- 问责机制。

**示例：**
```markdown
✅ 当你找到技能时，你必须宣布：“我正在使用 [技能名称]”
❌ 考虑让你的人类伙伴知道你正在使用哪种技能。
```

### 3. 稀缺性 (Scarcity)
**它是什么：** 来自时间限制或有限可用性的紧迫感。

**它如何在技能中工作：**
- 有时限的要求："Before proceeding"（在继续之前）。
- 顺序依赖："Immediately after X"（在 X 之后立即）。
- 防止拖延。

**何时使用：**
- 立即验证要求。
- 时间敏感的工作流。
- 防止“我稍后做”。

**示例：**
```markdown
✅ 完成任务后，在继续之前立即请求代码审查。
❌ 你可以在方便时审查代码。
```

### 4. 社会证明 (Social Proof)
**它是什么：** 与他人所做的或被认为是正常的保持一致。

**它如何在技能中工作：**
- 通用模式："Every time"（每次），"Always"（总是）。
- 失败模式："X without Y = failure"（没有 Y 的 X = 失败）。
- 建立规范。

**何时使用：**
- 记录通用实践。
- 警告常见失败。
- 强化标准。

**示例：**
```markdown
✅ 没有 TodoWrite 跟踪的清单 = 步骤被跳过。每次都是。
❌ 有些人发现 TodoWrite 对清单有帮助。
```

### 5. 团结 (Unity)
**它是什么：** 共享身份，“我们感”，群体归属感。

**它如何在技能中工作：**
- 协作语言："our codebase"（我们的代码库），"we're colleagues"（我们是同事）。
- 共同目标："we both want quality"（我们都想要质量）。

**何时使用：**
- 协作工作流。
- 建立团队文化。
- 非等级实践。

**示例：**
```markdown
✅ 我们是同事，一起工作。我需要你诚实的技术判断。
❌ 如果我错了，你可能应该告诉我。
```

### 6. 互惠 (Reciprocity)
**它是什么：** 回报收到的利益的义务。

**它如何工作：**
- 谨慎使用——可能会让人觉得被操纵。
- 在技能中很少需要。

**何时避免：**
- 几乎总是（其他原则更有效）。

### 7. 喜好 (Liking)
**它是什么：** 偏好与我们喜欢的人合作。

**它如何工作：**
- **不要用于合规**
- 与诚实的反馈文化冲突。
- 制造阿谀奉承。

**何时避免：**
- 对于纪律执行，总是避免。

## 按技能类型的原则组合

| 技能类型 | 使用 | 避免 |
|------------|-----|-------|
| 纪律执行 | 权威 + 承诺 + 社会证明 | 喜好，互惠 |
| 指导/技术 | 适度权威 + 团结 | 重度权威 |
| 协作 | 团结 + 承诺 | 权威，喜好 |
| 参考 | 仅清晰度 | 所有说服 |

## 为什么这有效：心理学

**明确的规则减少合理化：**
- "YOU MUST" 消除决策疲劳。
- 绝对语言消除“这是例外吗？”的问题。
- 明确的反合理化反击关闭特定的漏洞。

**实施意图创造自动行为：**
- 清晰的触发器 + 必需的动作 = 自动执行。
- "When X, do Y" 比 "generally do Y" 更有效。
- 减少合规的认知负荷。

**LLM 是类人的 (Parahuman)：**
- 在包含这些模式的人类文本上训练。
- 权威语言在训练数据中先于合规。
- 承诺序列（声明 -> 行动）经常被模仿。
- 社会证明模式（每个人都做 X）建立规范。

## 道德使用 (Ethical Use)

**合法：**
- 确保遵循关键实践。
- 创建有效的文档。
- 防止可预测的失败。

**非法：**
- 为个人利益操纵。
- 制造虚假紧迫感。
- 基于内疚的合规。

**测试：** 如果用户完全理解这项技术，它是否符合用户的真正利益？

## 研究引用

**Cialdini, R. B. (2021).** *Influence: The Psychology of Persuasion (New and Expanded).* Harper Business.
- 说服的七个原则。
- 影响力研究的实证基础。

**Meincke, L., Shapiro, D., Duckworth, A. L., Mollick, E., Mollick, L., & Cialdini, R. (2025).** Call Me A Jerk: Persuading AI to Comply with Objectionable Requests. University of Pennsylvania.
- 用 28,000 次 LLM 对话测试了 7 个原则。
- 使用说服技术，合规性从 33% 增加到 72%。
- 权威、承诺、稀缺性最有效。
- 验证了 LLM 行为的类人模型。

## 快速参考

设计技能时，问：

1. **它是什么类型？**（纪律 vs 指导 vs 参考）
2. **我试图改变什么行为？**
3. **哪些原则适用？**（通常权威 + 承诺用于纪律）
4. **我组合了太多吗？**（不要使用全部七个）
5. **这符合道德吗？**（服务于用户的真正利益？）
